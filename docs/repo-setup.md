# Repository Setup Notes

Эти шаги нужно выполнить один раз после поднятия репозитория в GitHub.

1. **Запрет прямых пушей в `main`.** Settings → Branches → Add rule. Включите
   "Require a pull request before merging" и запретите форс-пуши.
2. **Required check.** В той же форме отметьте workflow `CI` как обязательный.
3. **Добавьте CODEOWNERS** (см. пример ниже), если хотите автоматические запросы
   ревью с ведущего.
4. **Теги релизов.** После мержа ветки `p02/*` в `main`, создайте тег `P02` и
   дополните релиз заметками из pull request-а.

Пример `CODEOWNERS` для каталога идей:
```
# docs/ и app/ обслуживает продуктовая команда каталога
/docs/   @product-lead
/app/    @backend-lead
```

Сделайте скриншот настроек или сохраните вывод `gh api` для зачёта пункта C4.

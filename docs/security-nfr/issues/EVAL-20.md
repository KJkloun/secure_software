# EVAL-20 — Блокировка повторных оценок

## Контекст
Сейчас пользователь может отправить несколько оценок подряд и переписать голос без контроля. Для NFR-04 нужно ограничить активную оценку одним экземпляром на пользователя и идею.

## Что делаем
- хранить идентификаторы пользователей в списке оценок;
- возвращать 409, если попытка повторной оценки без обновления;
- обновить документацию по API оценок.

## Приёмка
1. Интеграционный тест `tests/security/test_eval_dedup.py` зелёный.
2. Лог сервиса фиксирует событие `duplicate_evaluation`.
3. BDD-сценарий для повторной оценки в `NFR_BDD.md` проходит.

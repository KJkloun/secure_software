# SEC-12 — Включить JWT-защиту административных эндпойнтов

## Контекст
Административная часть Idea Catalog API сейчас доступна без строгой проверки подписи. Для NFR-01 нужно включить проверку RS256 и ограничить TTL токена.

## Что делаем
- добавить проверку подписи RS256 в `/admin/*`;
- внедрить проверку истечения токена на уровне middleware;
- обновить переменные окружения (`AUTH_JWKS_URL`, `AUTH_ADMIN_AUD`).

## Приёмка
1. Контрактный тест `tests/security/test_admin_access.py` проходит на CI.
2. Запрос без токена возвращает 401.
3. Токсические кейсы из `docs/security-nfr/NFR_BDD.md` закрыты.

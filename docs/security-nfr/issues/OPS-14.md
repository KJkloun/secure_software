# OPS-14 — Ротация секретов через Vault

## Контекст
NFR-07 требует хранить секреты только во Vault/ENV и ротировать их каждые 30 дней. Сейчас часть токенов обновляется вручную раз в квартал.

## Что делаем
- настроить автоматическую ротацию секретов в Vault с TTL 30 дней;
- обновить runbook и добавить процедуру аварийной ротации;
- включить скрипт `scripts/secrets_ttl.py` в CI.

## Приёмка
1. Скрипт TTL завершается без ошибок и показывает срок < 30 дней.
2. Vault audit log содержит запись о плановой ротации.
3. В runbook зафиксирован новый процесс и контакты on-call.

# OPS-08 — Weekly pip-audit в CI

## Контекст
Согласно NFR-06, уязвимости High/Critical должны закрываться в течение семи дней. Для этого каждую неделю нужно запускать `pip-audit` и поднимать issue при обнаружении проблемы.

## Что делаем
- добавить шаг `pip-audit` в nightly workflow;
- сохранять отчёт в артефакты GitHub Actions;
- завести шаблон issue для найденных уязвимостей.

## Приёмка
1. Nightly job добавлена в `.github/workflows/security-nightly.yml`.
2. Первый отчёт `pip-audit` сохранён в артефактах и приложен к PR.
3. Для найденной уязвимости автоматически создаётся ticket с дедлайном 7 дней.

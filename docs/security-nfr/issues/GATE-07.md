# GATE-07 — Rate limiting в API Gateway

## Контекст
NFR-03 требует ограничить создание идей — не более 100 успешных POST `/ideas` на пользователя за минуту. Ограничение нужно реализовать на уровне gateway перед тем, как запрос попадёт в FastAPI.

## Что делаем
- настроить глобальный лимитатор в API Gateway;
- завести исключение для внутренних сервисных токенов;
- задокументировать конфиг в `docs/infra/rate-limit.md`.

## Приёмка
1. Нагрузочный тест `scripts/loadtest/ideas-rate-limit.js` возвращает 429 после 100 успешных запросов.
2. Лог gateway фиксирует события блокировки.
3. BDD-сценарий «Превышение лимита приводит к 429» проходит на stage.

# OBS-11 — Audit logging 401/403

## Контекст
Для NFR-05 нужно собирать почти все события 401/403 в отдельный security-лог и помечать их идентификатором клиента. Сейчас часть логов теряется в общих файлах приложения.

## Что делаем
- настроить отправку 401/403 в `security.log`;
- включить корелляционный `trace_id` в записи;
- обновить дашборд в Grafana (панель «Auth Failures»).

## Приёмка
1. Скрипт `scripts/check_security_logs.py` показывает покрытие ≥ 95%.
2. В дашборде видно рост метрик при тестовом сценарии.
3. Негативный BDD-сценарий про три подряд 401 выполняется на stage.

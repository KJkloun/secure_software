# OPS-16 — Шифрование бэкапов AES-256

## Контекст
NFR-08 требует хранить резервные копии приложения и базы только в шифрованном виде. Сейчас еженедельные бэкапы на S3 лежат без обязательной проверки алгоритма.

## Что делаем
- включить серверное шифрование AES-256 для бакета;
- завести автоматическую проверку ключа в KMS;
- обновить релизный чек-лист и runbook Disaster Recovery.

## Приёмка
1. Отчёт KMS подтверждает использование ключа `idea-catalog-backup`.
2. Скрипт проверки бэкапов возвращает «Encrypted: true».
3. Релизный чек-лист содержит пункт про шифрование и ссылку на отчёт.
